# Руководство пользователя SalesTracker

Документ описывает работу с приложением: привязку магазинов, синхронизацию
заказов и формирование отчётов. Сценарии соответствуют разделу 11.2 спецификации.

---

## 1. Главное окно

После запуска открывается одно окно с четырьмя вкладками:

| Вкладка | Назначение |
| --- | --- |
| **Настройки** | привязка и удаление магазинов |
| **Отчёт за день** | продажи по артикулам за одну дату |
| **Отчёт за период** | сводный отчёт за интервал дат |
| **Аналитика** | топ-артикулы, сравнение периодов, падение продаж |

Вверху окна расположены кнопка **«Синхронизировать»**, строка состояния
и счётчик заказов в базе данных.

---

## 2. Привязка магазина (вкладка «Настройки»)

### Wildberries
1. В блоке **Wildberries** введите название магазина (любое, для себя).
2. Вставьте **API-ключ** (категория «Статистика»).
3. Нажмите **«Сохранить ключ Wildberries»** — появится подтверждение.

### Ozon
1. В блоке **Ozon** введите название магазина.
2. Введите **Client-Id** и **Api-Key** из личного кабинета Ozon.
3. Нажмите **«Сохранить ключ Ozon»**.

Привязанные магазины отображаются в списке слева. Чтобы удалить магазин,
выберите его в списке и нажмите **«Удалить выбранный магазин»**
(вместе с магазином удаляются его заказы).

> API-ключи хранятся в локальной базе в закодированном виде и не передаются
> никуда, кроме официальных API маркетплейсов.

---

## 3. Синхронизация заказов

- **Вручную:** нажмите кнопку **«Синхронизировать»** вверху окна.
  Приложение загрузит заказы за последние 30 дней по всем магазинам.
  Ход выполнения отображается в строке состояния.
- **Автоматически:** пока приложение открыто, синхронизация запускается
  сама раз в час.

Повторная синхронизация не создаёт дубликатов — уже загруженные заказы
обновляются по их уникальному идентификатору.

Если API недоступен или ключ неверный, появится окно с текстом ошибки,
а ранее загруженные данные сохранятся.

---

## 4. Фильтр по магазинам

На вкладках с отчётами есть блок **«Магазины»**. Выберите один или несколько
магазинов, чтобы построить отчёт только по ним. Если ничего не выбрано —
отчёт строится по всем магазинам.

---

## Сценарий A. Просмотреть отчёт за день (роль «Селлер»)

1. Откройте вкладку **«Отчёт за день»**.
2. Укажите дату в формате `ГГГГ-ММ-ДД` (по умолчанию — сегодня).
3. При необходимости выберите магазины в фильтре.
4. Нажмите **«Построить отчёт»**.
5. В таблице появятся артикулы с количеством заказов. Столбцы можно
   сортировать кликом по заголовку.

---

## Сценарий B. Сформировать отчёт за период (роль «Менеджер»)

1. Откройте вкладку **«Отчёт за период»**.
2. Укажите даты **«С»** и **«По»** (период от 1 до 90 дней).
3. Выберите магазины (необязательно) и нажмите **«Построить отчёт»**.
4. Отчёт показывает разбивку по дням и артикулам.

---

## Сценарий C. Выявить топ-артикулы и сравнить периоды (роль «Аналитик»)

1. Откройте вкладку **«Аналитика»**.
2. В списке **«Тип анализа»** выберите нужный режим:
   - **Топ артикулов** — самые продаваемые артикулы за Период 1.
     Число артикулов задаётся полем **N** (от 5 до 50).
   - **Сравнение периодов** — таблица с колонками «Период 1», «Период 2»,
     «Δ количество», «Δ %». Периоды должны быть одинаковой длительности.
   - **Отрицательная динамика** — только артикулы, продажи которых упали,
     отсортированные по величине падения в процентах.
3. Заполните даты периодов и нажмите **«Построить»**.

---

## 5. Экспорт отчёта

Под любой таблицей есть кнопки:

- **«Экспорт в XLSX»** — сохранить отчёт в файл Excel;
- **«Экспорт в CSV»** — сохранить отчёт в текстовый CSV (разделитель `;`).

Выберите папку и имя файла в диалоге сохранения. После сохранения
появится сообщение с полным путём к файлу.

---

## Частые вопросы

**Отчёт пустой.** Сначала привяжите магазин и нажмите «Синхронизировать».
Убедитесь, что за выбранную дату/период действительно были заказы.

**Ошибка при синхронизации.** Проверьте интернет и правильность ключей.
Для Ozon обязательно нужны и Client-Id, и Api-Key.

**Где лежат данные.** В файле `data/salestracker.mv.db` рядом с приложением.
Этот файл можно скопировать для резервной копии.
